# Drone Intercept Simulator — 3D

Браузерная симуляция перехвата ударных дронов антидронами с полной 3D аэродинамикой.  
Поле 3000 × 3000 м, вид сверху + вид сбоку (высотный срез).

## Запуск

```
npm install
npm run dev
```

Открыть:

- `http://localhost:5173` — симулятор
- `http://localhost:5173/whitepaper.html` — whitepaper с формулами и печатью в PDF

## Физическая модель

### Две модели (`src/physics.js`)

Дрон и антидрон считаются разными аппаратами.

- **Дрон** — тяжёлый самолёт/fixed-wing. Висения нет: тяга только вперёд, скорость не командуется ниже `stallSpeed`, набор высоты возможен только за счёт запаса скорости. Если скорость падает к сваливанию, вертикальный канал не может удерживать высоту.
- **Антидрон** — лёгкий мультикоптер. Горизонталь и вертикаль делят общий 3D-вектор тяги, гравитация явная, висение физически допустимо.
- **Разворот** — у обоих ограничен актуатором и центростремительным ускорением, поэтому на высокой скорости радиус разворота растёт.
- **Сопротивление** — квадратичное по скорости. У дрона оно тормозит поступательное движение, у антидрона входит в общий расчёт тяги.
- **Тяговооружённость**: дрон ×0.35 по прямой тяге, антидрон ×3.0.

### Параметры по умолчанию

| Параметр | Дрон | Антидрон |
|---|---|---|
| Масса | 15 кг | 0.5 кг |
| Скорость | 200 км/ч | 350 км/ч |
| Скорость сваливания | 65 км/ч | — |
| Инерция (responseTime) | 0.55 с | 0.22 с |
| Макс. верт. скорость | 12 м/с | 20 м/с |
| Угловая скорость | 45 °/с | 90 °/с |
| Высотный пол | мягкий пол 15 м через целевую высоту `tz ≥ 15 м` до вызова физики (снимается в `dive`) | мягкий пол 15 м через целевую высоту `gz ≥ 15 м` (снимается только в терминальной фазе перехвата ≤ 20 м от цели) |

Если активный дрон или антидрон оказался ниже **1 м**, он считается врезавшимся в землю (`mode = crashed`). `base`/`waiting` у антидрона стоят на земле и не считаются полётом.

### Порядок шага симуляции

Каждый тик (`DT = 0.02 с`) строго соблюдает порядок:

1. **Принятие решения / предсказание траектории** — политика дрона или алгоритм преследования вычисляют целевую точку, опережающий прицел и ограничения (высотный пол `gz`).
2. **Физика** — дрон: `navigateFixedWingToPoint`; антидрон: `navigateToPoint` → `applyPhysics`. На этом шаге обновляются скорости и позиция.
3. **Постфизика** — только событийные проверки: попадание в цель после пике, столкновение с землёй (`< 1 м`), взрыв при перехвате.

Ограничения высотного пола применяются на шаге 1 (через `gz`), а не как коррекция позиции после физики.

### Нос-вперёд и разворот

Тяга/скорость всегда считаются по оси рыскания (`targetVx = cos(angle) × speed`).  
Скорость разворота ограничена двумя условиями одновременно:

```
effectiveYawRate = min(actuator_yawRate, maxAccH / currentSpeed)
```

При высокой скорости — центростремительное ограничение (реалистичный радиус разворота).  
При малой скорости — ограничение актуатора.

**Снижение скорости в развороте:**

```
alignFactor = max(minSpeedFactor, (1 + cos(yawError)) / 2)
```

Антидрон (`minSpeedFactor = 0.18`) остаётся манёвренным, но не тормозит почти до зависания на 180° развороте.  
Дрон (`minSpeedFactor = 0.65`) не может тормозить как коптер: команда скорости держится выше 65 % крейсерской скорости и не ниже `stallSpeed`.

## Сценарий атаки дрона

Три фазы (`src/policies/dronePolicy.js`):

### 1. Крейсерский полёт (> 500 м до цели)
- Высота 30 м (CRUISE_ALT)
- Уклонение от антидронов — обнаружение в сфере 600 м
- Если угроза: уход на эвейд-вейпоинт. Высота уклонения выбирается относительно угрозы: обычно 70 м или 15 м, и фиксируется на весь текущий `evade`, чтобы не было дрожания 15/70.

### 2. Набор высоты (< 500 м до цели)
- Уклонение отключается, дрон идёт прямо на цель
- Целевая высота: **50 м** (ATTACK_ALT)
- Скорость: текущая максимальная скорость дрона из `dspeed`
- Если заход уже начался (`climb` или `dive`), дрон больше не уворачивается и не ретаргетится на новую цель.
- Перед входом в `climb` rule-based политика проверяет выполнимость захода: курс на цель, дистанцию на набор высоты, дистанцию на последующее снижение и запас на разворот.
- Если дистанции или курса не хватает, дрон не начинает `climb`, а уходит в `reapproach`: пролетает вперёд на 250 м или примерно 3 радиуса разворота и пробует зайти снова.
- Во время `climb` окно пикирования проверяется повторно: если дрон ещё высоко, а дистанции до цели уже не хватает для чистого снижения, он уходит на повторный заход вместо позднего пике в землю.
- Эта проверка находится только в `RuleBasedDronePolicy`; физика и симулятор не запрещают ML-политике выбирать собственный заход.

### 3. Пикирование (высота ≥ 45 м)
- Целевая высота: **0 м**
- Скорость: текущая максимальная скорость дрона из `dspeed`
- Высотная безопасность отключена; если промахнулся и ушёл ниже 1 м — `crashed`
- Цель поражена после физического шага, когда горизонтальное расстояние < 25 м и высота ≤ 5 м

## Антидрон — конечный автомат

```
base → waiting → intercept → chase → lastknown → recoverSearch → circle
                                  ↑         ↓
                                  └─────────┘  (обнаружил снова)
```

| Состояние | Поведение |
|---|---|
| `base` | Стоит, ждёт команды камеры |
| `waiting` | Задержка вылета (`adelay`, дефолт 5 с, диапазон 0–30 с) |
| `intercept` | Летит к последней известной позиции |
| `chase` | Упреждающее преследование — навигация к предсказанной точке |
| `lastknown` | Летит к точке потери цели |
| `recoverSearch` | После потери цели набирает высоту к 45 м с малой поступательной скоростью |
| `circle` | Барражирует кругами на рабочей высоте поиска 45 м |

При потере визуального контакта из `chase` антидрон не уходит в `circle` мгновенно: есть инерциальное сопровождение **3.0 с** по последней известной позиции и скорости цели. Это нужно потому, что при упреждении нос антидрона может смотреть в расчётную точку перехвата, а сам дрон на подлёте оказывается на краю конуса.

**Упреждающее преследование** (`chase`) считает точку перехвата для цели с постоянной скоростью:

```
tLead = positive root of |relativePosition + targetVelocity × t| = anti.maxSpeed × t
tLead = clamp(tLead, 0, 2.5)
aimPoint = drone.pos + drone.vel × tLead
```

Если аналитического корня нет, используется fallback `dist3D / anti.maxSpeed`. Перехват проверяется до и после физического шага антидрона, чтобы не пропускать пересечение сферы между тиками.

**Перехват**: 3D расстояние < 20 м. Это радиус proximity fuse, а не требование геометрического касания центров.

**Барометр**: в дальнем `chase` целевая высота `gz` клемпируется к 15 м *до* вызова физики — это аварийный пол без скачка позиции. В `lastknown` и `circle` антидрон уходит выше мачт камер: рабочая высота поиска **45 м**. Если `circle` начинается ниже 40 м, антидрон набирает высоту с 25 % горизонтальной скорости, чтобы не зависать и не кружить на высоте мачт. Барометрическая безопасность отключается только в терминальной фазе, когда антидрон уже рядом с живой целью по горизонтали (`≤ INTERCEPT_R`). Если антидрон всё же ушёл ниже 1 м — `crashed`.

## Система обнаружения

### Камеры обнаружения (10 штук)

- Высота мачты: **15 м**
- Горизонтальный FOV: **50°** (±25°)
- Дальность: 900 м (регулируется)
- Проверка: `inCone3d` — 3D дальность + горизонтальный пеленг + |Δz| < 200 м
- При обнаружении: запуск ближайшего свободного антидрона с задержкой

### Антидрон — бортовая камера

- Горизонтальный FOV: **55°** (±27.5°)
- Дальность: 650 м (регулируется)
- **Конус наклоняется с антидроном**: `pitch = atan2(vz, vH)`
- Проверка: `inCone3dFull` — настоящий 3D конус через dot product

```
axis = (cos(yaw)·cos(pitch), sin(yaw)·cos(pitch), sin(pitch))
visible = dot(axis, toTarget) / dist ≥ cos(FOV / 2)
```

### Дрон — радар угрозы

- **Сфера 600 м**, всенаправленная
- Проверка: `distance3d(drone, anti) ≤ DRONE_DETECT_R`

## Интерфейс

**Вид сверху** — карта 3000 × 3000 м  
- Колесо мыши — зум  
- ПКМ + drag — панорама  
- Цвет следа дрона: зелёный (низко) → голубой (высоко)

**Вид сбоку** (под картой) — срез X / Altitude  
- Горизонталь синхронизирована с зумом и панорамой верхнего вида
- Вертикальная шкала ограничена **0–120 м**
- Камеры на мачтах 15 м (синие точки), цели на земле (оранжевые кресты)
- Пунктир на 30 м — крейсерская высота
- Это пространственная проекция `X / Altitude`: при разворотах в плане траектории могут накладываться друг на друга, поэтому горизонтальный манёвр нужно сверять с верхним видом.

**Правая панель** — справочная сводка по физике и FSM антидрона.

## Что осталось от модели обучения

Сейчас в рабочем приложении используется только `RuleBasedDronePolicy`. Полноценной ONNX/ML-политики в коде нет, переключатель политик из интерфейса удалён.

Оставшийся задел:

- `Simulation.setDronePolicy(policy)` и контракт actions в `applyDroneAction()` позволяют подменить rule-based политику внешним агентом.
- `RuleBasedDronePolicy` остаётся baseline-политикой для сравнения с будущим ML.
- `whitepaper.html` содержит roadmap обучения: observation, action space, reward, curriculum и метрики.
- `src/charts.js` содержит старый/запасной код overlay для `results.json` и `data/train_log.json`, но сейчас он не импортируется в `app.js` и не виден в интерфейсе.

Чтобы снова включить обучение как рабочий контур, нужно добавить runner/trainer, формат сохранения `results.json`/`train_log.json`, UI-кнопку результатов и реальную ML-политику, которая реализует тот же action contract, что и `RuleBasedDronePolicy`.
