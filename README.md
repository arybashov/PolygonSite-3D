# Drone Intercept Simulator — 3D

Браузерная симуляция перехвата ударных дронов антидронами с полной 3D аэродинамикой.  
Поле 3000 × 3000 м, вид сверху + вид сбоку (высотный срез).

## Запуск

```
npm install
npm run dev
```

Открыть `http://localhost:5173`.

---

## Физическая модель

### Многороторный коптер (`src/physics.js`)

Модель первого порядка: скорость отслеживает команду через P-регулятор с насыщением тяги.

- **Горизонталь** — 80 % тяги, квадратичное аэродинамическое сопротивление  
  `dragCoef = 0.80 × maxThrust / maxSpeed²` — равновесие точно на `maxSpeed`
- **Вертикаль** — 20 % тяги, декаплировано от горизонтального канала  
  `targetVz = clamp((tz − z) × 3.0, −maxVz, maxVz)`
- **Снижение** — вниз ускорение может быть выше за счёт гравитации: ограничение берётся как `max(0.20 × maxThrust / mass, 9.81)`
- **Гравитация** — скомпенсирована коллективным газом (hover assumption)
- **Тяговооружённость**: дрон ×1.6, антидрон ×3.0

### Параметры по умолчанию

| Параметр | Дрон | Антидрон |
|---|---|---|
| Масса | 15 кг | 0.5 кг |
| Скорость | 200 км/ч | 350 км/ч |
| Скорость сваливания | 65 км/ч | — |
| Инерция (responseTime) | 0.55 с | 0.10 с |
| Макс. верт. скорость | 12 м/с | 20 м/с |
| Угловая скорость | 45 °/с | 90 °/с |
| Высотный пол | 15 м (снимается в финальном `dive`) | 15 м (снимается только в терминальной фазе перехвата рядом с целью) |

Если активный дрон или антидрон оказался ниже **1 м**, он считается врезавшимся в землю (`mode = crashed`). `base`/`waiting` у антидрона стоят на земле и не считаются полётом.

### Нос-вперёд и разворот

Тяга всегда по оси рыскания (`targetVx = cos(angle) × speed`).  
Скорость разворота ограничена двумя условиями одновременно:

```
effectiveYawRate = min(actuator_yawRate, maxAccH / currentSpeed)
```

При высокой скорости — центростремительное ограничение (реалистичный радиус разворота).  
При малой скорости — ограничение актуатора.

**Снижение скорости в развороте:**

```
alignFactor = max(minSpeedFactor, (1 + cos(yawError)) / 2)
```

Антидрон (`minSpeedFactor = 0.05`) тормозит почти до нуля на 180° развороте.  
Дрон (`minSpeedFactor = 0.30`) не падает ниже 30 % круизной скорости плюс stallSpeed.

## Сценарий атаки дрона

Три фазы (`src/policies/dronePolicy.js`):

### 1. Крейсерский полёт (> 500 м до цели)
- Высота 30 м (CRUISE_ALT)
- Уклонение от антидронов — обнаружение в сфере 600 м
- Если угроза: уход на эвейд-вейпоинт. Высота уклонения выбирается относительно угрозы: обычно 70 м или 15 м, и фиксируется на весь текущий `evade`, чтобы не было дрожания 15/70.

### 2. Набор высоты (< 500 м до цели)
- Уклонение отключается, дрон идёт прямо на цель
- Целевая высота: **50 м** (ATTACK_ALT)
- Скорость: текущая максимальная скорость дрона из `dspeed`
- Если заход уже начался (`climb` или `dive`), дрон больше не уворачивается и не ретаргетится на новую цель.
- Перед входом в `climb` rule-based политика проверяет выполнимость захода: курс на цель, дистанцию на набор высоты, дистанцию на последующее снижение и запас на разворот.
- Если дистанции или курса не хватает, дрон не начинает `climb`, а уходит в `reapproach`: пролетает вперёд на 250 м или примерно 3 радиуса разворота и пробует зайти снова.
- Эта проверка находится только в `RuleBasedDronePolicy`; физика и симулятор не запрещают ML-политике выбирать собственный заход.

### 3. Пикирование (высота ≥ 45 м)
- Целевая высота: **0 м**
- Скорость: текущая максимальная скорость дрона из `dspeed`
- Высотная безопасность отключена; если промахнулся и ушёл ниже 1 м — `crashed`
- Цель поражена, когда горизонтальное расстояние < 25 м и высота ≤ 5 м

## Антидрон — конечный автомат

```
base → waiting → intercept → chase → lastknown → circle
                                  ↑         ↓
                                  └─────────┘  (обнаружил снова)
```

| Состояние | Поведение |
|---|---|
| `base` | Стоит, ждёт команды камеры |
| `waiting` | Задержка вылета (adelay, 0–30 с) |
| `intercept` | Летит к последней известной позиции |
| `chase` | Упреждающее преследование — навигация к предсказанной точке |
| `lastknown` | Летит к точке потери цели |
| `circle` | Барражирует кругами, ищет дроны |

При потере визуального контакта из `chase` антидрон не уходит в `circle` мгновенно: есть grace-период **0.35 с**, чтобы не было дрожания режимов на границе конуса видимости.

**Упреждающее преследование** (`chase`):

```
tLead = min(2.5, dist3D / anti.maxSpeed)
aimPoint = drone.pos + drone.vel × tLead
```

**Перехват**: 3D расстояние < 45 м.

**Барометр**: в поиске, `lastknown`, `circle` и дальнем `chase` антидрон держит высотный пол 15 м. Барометрическая безопасность отключается только в терминальной фазе, когда антидрон уже рядом с живой целью по горизонтали (`≤ INTERCEPT_R`); если промахнулся и ушёл ниже 1 м — `crashed`.

## Система обнаружения

### Камеры обнаружения (10 штук)

- Высота мачты: **15 м**
- Горизонтальный FOV: **50°** (±25°)
- Дальность: 900 м (регулируется)
- Проверка: `inCone3d` — 3D дальность + горизонтальный пеленг + |Δz| < 200 м
- При обнаружении: запуск ближайшего свободного антидрона с задержкой

### Антидрон — бортовая камера

- Горизонтальный FOV: **55°** (±27.5°)
- Дальность: 650 м (регулируется)
- **Конус наклоняется с антидроном**: `pitch = atan2(vz, vH)`
- Проверка: `inCone3dFull` — настоящий 3D конус через dot product

```
axis = (cos(yaw)·cos(pitch), sin(yaw)·cos(pitch), sin(pitch))
visible = dot(axis, toTarget) / dist ≥ cos(FOV / 2)
```

### Дрон — радар угрозы

- **Сфера 600 м**, всенаправленная
- Проверка: `distance3d(drone, anti) ≤ DRONE_DETECT_R`

## Интерфейс

**Вид сверху** — карта 3000 × 3000 м  
- Колесо мыши — зум  
- ПКМ + drag — панорама  
- Цвет следа дрона: зелёный (низко) → голубой (высоко)

**Вид сбоку** (под картой) — срез X / Altitude  
- Горизонталь синхронизирована с зумом и панорамой верхнего вида  
- Вертикальная шкала ограничена **0–120 м**
- Камеры на мачтах 15 м (синие точки), цели на земле (оранжевые крресты)  
- Пунктир на 30 м — крейсерская высота
- Это именно проекция `X / Altitude`, а не профиль по пройденной дистанции. При разворотах в плане линия может выглядеть с изломами даже при плавной 3D-траектории.

**Правая панель** — переключение политики дрона: Rule-based / ONNX
